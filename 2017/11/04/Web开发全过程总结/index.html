事件委托模式

工作原理
因为click和keypress等事件都会通过DOM传播，这就意味着他们的祖先也会接收到事件。每当要给待处理清单这类动态添加或者删除的元素绑定事件时，都应该使用事件委托的方式，为动态的添加元素的容器添加单一事件监听程序，然后根据实际触发事件的元素执行相应的处理程序是更为容易，更加高效的方式。

表单校验方式

1.在HTML上添加校验属性，使用浏览器内置的校验机制进行校验。
2.使用约束校验API，通过js编写校验的代码

required

检测表单字段是否为空，这种校验对于带有默认值的字段不会生效。

pattern

限制表单字段的数据格式，配合正则表达式(用于模式匹配的字符)。

约束校验API

regexObj.test(str)
str用来与正则表达式匹配的字符串
在用户输入内容的时候进行校验是最佳校验时间

AJAX

1.提取用户输入的数据，然后使用js更新表格和清单
ajax的核心XMLHttpRequest API。通过XMLHttpRequest对象，可以在监听请求，响应周期中的任何阶段。其中response属性包括了服务器返回的数据（HTML、XML、JSON），status是用于展示HTTP响应是否成功的数字代码，也被称为HTTP状态码。
