跨域

是指浏览器不能执行其他网站的脚本，有同源策略引起是浏览器对JS实施的安全限制
同源策略的限制行为有：

  • Cookie、LocalStorage和indexDB无法读取
  • DOM和JS对象无法获取
  • Ajax请求发不出去
    所谓的同源就是指域名，协议，得为均为相同
    #跨域的解决办法
    jsonp跨域
    在html页面中通过相应的标签从不同域名下加载静态资源文件是被浏览器允许的。
    原生代码

利用jquery。但是缺陷也很明显，只能实现GET请求。

在主域名相同的情况下使用 document.domain + iframe。
window.name + iframe以及location.hash + iframe。

WebSocket

HTTP协议的缺陷（通信只能由客户端发起）Websocket就是为了解决轮询带来的问题。
WebSocket的最大特点就是服务器可以主动向客户端推送信息，客户端也可以向服务器发送信息，是真正的双向平等对话。
1、客户端API
1.1 WebSocket构造函数

1.2 webSocket.readyState
readyState属性返回实例对象的当前状态

connecting: 0 表示正在连接
open: 1 表示连接成功了，可以通信了
closing: 2 表示连接正在关闭
closed: 3 表示连接已经关闭，或者打开连接失败。
1.3 webSocket.onopen
用于指定连接成功后的回调函数，如果要指定多个回调函数，可以使用addEventListener方法。
1.4 webSocket.onclose
用于指定连接关闭后的回调函数，同样可以使用addEventListener。
1.5 webSocket.onmessage
用于指定收到发服务器数据后的回调函数。服务器数据可能是文本，也可能是二进制数据（blob对象或Arraybuffer对象）。
除了动态判断收到的数据类型，也可以使用binaryType属性，显示指定收到的二进制数据类型。
1.6 webSocket.send()
实例对象的send()方法用于向服务器发送数据。
1.7 webSocket.bufferedAmount
表示还有多少个字节的二进制数据没有发送出去，可以用它来判断发送时否结束。
1.8 webSocket.onerror
用于指定报错时的回调函数
