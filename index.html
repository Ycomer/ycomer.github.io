1、parser解析是在浏览器所以有性能问题，首屏渲染卡顿

块级元素的基本特征：
块级元素的流体特征主要体现在水平方向上。

外部尺寸与流体的特性

1.正常流宽
要充分利用block容器的流特性，一旦为表现为外部尺寸的块级元素设置了高度，流动性（margin/border/padding/content内容区域自动分配水平空间的机制）就丢失了。
2.格式化宽度
格式化宽度仅仅出现在绝对定位模型中，也就是position为absolute或fixed的元素中。绝对定位元素的宽度由内部尺寸决定，对于非替换元素，当left/top/bottom/对方方位的属性值同时存在的时候，元素的宽度表现为格式化宽度。其宽度大小相对于最近的具有定位特性的祖先元素计算。

内部尺寸与流体特性

1.包裹性
自适应性指的是元素尺寸由内部元素决定，但永远小于包含快容器的尺寸。除了inline-block元素，浮动元素以及绝对定位的元素都具有包裹性，均有类似的智能宽度行为。
2.首选最小宽度
元素最合适的最小宽度。
3.最大宽度
元素可以有的最大宽度。实现自定义滚动有两种原理，一种借助原生的滚动，scroolLeft/scrollTop值的变化，另外一种是根据元素的尺寸和容器的关系，通过修改内部元素的位置实现滚动效果。

什么是Event loop

发表于 2017-12-26 | 阅读次数

JS单线程

JS语言的核心就是单线程，为了利用CPU的计算能力，HTML5提出Web Worker，运行js创建多个线程。但是字线程完全受主线程控制，且不得操作DOM。

任务队列

单线程意味着所有任务需要排队，前一个任务结束，下一个任务才能开始。因为io设备的原因所有任务可以分为两种一种是同步任务，一种是异步任务。异步任务不进入主线程，而进入任务队列。任务队列的的任务通知主线程某个异步任务可以执行了，改任务才会进入主线程。

异步执行的机制

1.所有同步任务都在主线程上执行，形成执行栈。
2.主线程之外还有一个任务队列，主要异步任务有了运行结果，就在任务队列中放置一个事件
3.一旦执行栈中的同步任务完成，系统就会读取任务队列，然后将异步任务加入执行栈执行。
4.主线程不断重复上述的第三步。

事件和回调函数

任务队列就是一个事件的队列
回调函数就是那些被主线程挂起来的代码，异步任务必须指定回调函数，主线程开始执行异步任务就是执行的回调函数。任务队列是一个先进先出的数据结构。

Event Loop

主线程从任务列队中读取事件，这个过程是循环不断的，又称Event Loop（事件循环），主线程在运行的时候，产生堆和栈，栈中的代码调用各种外部的API，它们在任务队列中加入各种事件，只要栈中的代码执行完毕，主线程就会去读取任务列队，依次执行那些事件所对应的回调函数。执行栈的代码，总是在读取任务队列之前执行。

定时器

定时器主要由setTimeut()和setInterval()这两个函数完成。它们的内部运行机制一致，区别在于前者指定的代码是一次性执行，后者则是反复执行。
setTimeout(fn, 0)接受两个参数，第一个是回调函数，第二个是推迟执行的毫秒数。指定某个人物在主线程最早的空闲时间执行，也就是说尽可能早的执行，它在任务队列的尾部添加一个事件，因此要等到同步任务和同步任务都完成以后才会得到处理。

Node.js的Event Loop

Node.js也是单线程的Event loop，但是它的运行机制不同于浏览器环境。
Node.js的运行机制
1.V8引擎解析JS脚本
2.解析后的代码调用Node API
3.libuv库负责Node API的执行，它将不同的任务分配给不同的线程，形成一个Event loop，以异步的方式将任务的执行结果返回给V8引擎。
4.V8引擎再将结果返回给用户
除了浏览器中定时器的用法外，Node.js还有process.NextTick和setImmediate。首先process.nextTick方法可以在当前执行栈的尾部，下一次Event Loop之前，触发回调函数，也就是说，它指定的任务总是发生在所有异步任务之前。setImmediate方法则是在当前任务队列的尾部添加事件，也就是说，它指定的任务总是在下一次Event Loop时执行。

  • 多个process.nextTick语句总是在当前执行栈一次执行完，多个setImmediate可能则需要多此loop才能执行完。

JS开发者的十大重要之事

1.掌握语言的核心
一门语言的核心和标准库实际上是非常小的，掌握了其中的核心再加上其的生态系统是一个不错的开始。关于JS的语言的核心比如：if/else， 错误处理（throw/try/catch)，数组的操作（push/map/reduce/filter)，字符串处理（replace/slice/split），真假的处理，JSON对象的处理方法，以及变量作用域在js中如何工作，还有ES6中加入的特性（Promises/modules/classes/template literals/generators/destructuring）
2.理解异步代码
回调函数（callback）和Promise,event loop,学习新的async-await或者RxJS。
3.熟练的掌握一套开发者工具
解决bug的能力和理解系统的工作原理。会使用dev tool打断点，使用观察模式和利用控住台测试语句和堆栈跟踪，利用元素面板操作DOM以及网络面板debug网络请求。
4.精通JS函数
理解函数作用域，闭包，this，标准函数和箭头函数的区别，知道如何使用诸如map/filter/reduce之类的方法来处理数组函数。
5.能解决级基本的设计任务
会通过CSS和HTML来构建UI。
6.理解基本的网络和HTTP APIS
知道当在浏览器中输入URL是会发生什么。熟知HTTP请求，以及什么是REST和AJAX。知道JS中HTTP请求的用法（XMLHttpRequest,fetch,jQuery’s ajax)。
7.理解Node.js
会使用node以及npm等包管理器。熟练使用webpack等工具。
8.会使用框架来构建应用
熟练掌握其中一个或多个框架（React Vue Angular），在构建任务中根据自己的项目需求选择一个合适的框架
9.了解JS程序性能的基础知识
了解如何调试缓慢的流程，并了解问题的出处。了解解析缓慢的代码是由于运行慢还是网络延迟。学习如何保持ui响应并且不用其他计算来阻止渲染。理解如何缓存数据避免向服务器提出太多请求。学习计算机科学的相关知识，数据结构和算法。
10.持续的学习力
持续关注行业内容。
