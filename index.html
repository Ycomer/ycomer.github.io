先说RESTful API的缺点

一、扩展性
随api的不断发展，会变得越来越臃肿。
server返回的用户信息越来越多，client需要的信息只是其中的一部分，这样会增加网络传输量。并且不利于client处理数据。
调用多个api才能满足不同的数据需求，导致需要调用不同的api很是不灵活。

GraphQL

简单来说就是取那些数据是由客户端说了，在REST中给，给那些数据是server决定的，client只能从中挑选。在GraphQL中client直接对server说想要什么数据，server就负责返回什么数据。

文档

文档是mongDB的核心，类似于SQLite数据库关系数据库中的每一行。多个键及其关联的值放在一起就是文档。

文档的逻辑关系

1.嵌入式关系
2.引用式关系，将两个文档分开，通过引用文档的_id
字段来建立关系。

集合

集合就是一组文档的集合，就相当于关系数据库中的表一样。
1.元数据
数据库的信息存储在集合中，他们使用统一的命名空间DBNAME.system.*

常用命令

use dbname 创建数据库
db 查看当前数据库
show dbs 查看所有数据库
db.dropDatabase() 销毁数据库
createCollection()：创建集合
db.COLLECTION.drop()：删除集合
db.COLLECTION_NAME.insert(document)：插入文档
db.COLLECTION_NAME.update(SELECTION_CRITERIA,UPDATED_DATA)：更新文档
db.COLLECTION_NAME.save({_id:ObjectId(),NEW_DATA})：替换已存在的文档
db.COLLECTION_NAME.remove(DELECTION_CRITERIA)：删除文档

查询语句

当find中传入多个键值对时，MongoDB就会将其作为AND处理
db.users.find() 查询数据，不带参数返回所有数据。
db.users.find().pretty() 可以使查询的结果更为美观。
OR处理方法

db.post.find(
{
$or: [
{key1: value1}, {key2:value2}
]
}
).pretty()
PS : {$gt:10} 表示大于10，另外，$lt 表示小于，$lte 表示小于等于，$gte 表示大于等于，$ne 表示不等于。

文档操作

创建集合 db.createCollection(“shiyanlou2”, { capped : 1, autoIndexID : 1, size : 6142800, max : 10000 } ) #带参数
{ “ok “: 1 }
参数描述：

  • capped：类型为 Boolean，如果为 true 则创建一个固定大小的集合，当其条目达到最大时可以自动覆盖以前的条目。在设置其为 true 时也要指定参数大小；
  • autoIndexID：类型为 Boolean，默认为 false，如果设置为 true，则会在 _id field.s 上自动创建索引；
  • size：如果 capped 为 true 需要指定，指定参数的最大值，单位为 byte；
  • max：指定最大的文档数。 在 Mongodb 中也可以不用创建集合，因为在创建文档的时候也会自动的创建集合。

    查询索引和聚合

    1.条件操作符
    MongoDB 中的条件操作符有：
    (>) 大于 - $gt #greater than
    (<) 小于 - $lt #less than
    (>=) 大于等于 - $gte #equal
    (<= ) 小于等于 - $lte
    2.条件操作符
    $type
    3.limit与skip
    读取指定数量的数据记录 limit
    读取时跳过指定数量的数据记录 skip
    当然，还可以添加 find 的查找条件的参数，以便进行更精确的查找。
    4.排序sort()
    升序用1表示，降序用-1表示
    5.索引ensureIndex()
    索引是特殊的数据结构，索引存储在一个易于遍历读取的数据集合中，索引是对数据库集合中一个文档或多个文档的值进行排序的一种结构。
    6.聚合 aggregate()

实例：db.shiyanlou.aggregate([{$group:{_id:”$name”, user:{$sum:”$user_id”}}}])
7.管道
MongoDB 的聚合管道将 MongoDB 文档在一个管道处理完毕后将结果传递给下一个管道处理。管道操作是可以重复的。
表达式：处理输入文档并输出。表达式是无状态的，只能用于计算当前聚合管道的文档，不能处理其它的文档。
聚合框架中常用的几个操作
$project：修改输入文档的结构。可以用来重命名、增加或删除域，也可以用于创建计算结果以及嵌套文档。
$match：用于过滤数据，只输出符合条件的文档。$match 使用 MongoDB 的标准查询操作。
$limit：用来限制 MongoDB 聚合管道返回的文档数。
$skip：在聚合管道中跳过指定数量的文档，并返回余下的文档。
$unwind：将文档中的某一个数组类型字段拆分成多条，每条包含数组中的一个值。
$group：将集合中的文档分组，可用于统计结果。
$sort：将输入文档排序后输出。
$geoNear：输出接近某一地理位置的有序文档。

高级索引和查询

为了验证我们使用了索引，可以使用 explain 命令
1.原子操作
所谓原子操作，就是要么执行成功，要么执行失败，执行成功完成既定任务，执行失败还原执行前的状态。
常用原子操作命令：
1) $set
用来指定一个键并更新键值，若键不存在并创建。
2) $unset
用来删除一个键。
3) $inc
$inc 可以对文档的某个值为数字型（只能为满足要求的数字）的键进行增减的操作。
4) $push
把 value 追加到 field 里面去，field 一定要是数组类型才行，如果 field 不存在，会新增一个数组类型加进去。
5) $pushAll
同 $push ,只是一次可以追加多个值到一个数组字段内。
6) $pull
从数组 field 内删除一个等于 value 值。
7) $addToSet
增加一个值到数组内，而且只有当这个值不在数组内才增加。
8) $pop
删除数组的第一个或最后一个元素。
9) $rename
修改字段名称
10) $bit
位操作，integer 类型
2.查询分析explain()
explain 操作提供了查询信息，使用索引及查询统计等。有利于我们对索引的优化
3.hint()
虽然 MongoDB 查询优化器一般工作的很不错，但是也可以使用 hints 来强迫 MongoDB 使用一个指定的索引。
通过这种方法在某些情形下会提升性能。

前后端模板引擎的思路

发表于 2017-10-12 | 阅读次数

后端模板

在后端mvc中，一般从 Model 层中读取数据，然后将数据传到 View 层渲染（渲染成 HTML 文件），而 View 层，一般都会用到模板引擎。
以前的后端大部分会采用mvc模式。优点是有利于seo和减少http请求。但是缺点就很明显了。在前端更改内容的同时后端的模板引擎也会更改。那么问题来了，前端中的有大量的js是前端写还是后端，前端写没有数据调试不便，后端写不一定懂js。

前端模板

特点是依赖于客户端，在浏览器渲染页面，而不依赖于服务端。
归功于AJAX的出现前后端分离变成可能，后端专注业务逻辑，而前端提供接口。然后前端通过AJAX向后端请求数据，动态渲染页面。

模板引擎

用以生成html等多种文件
客户端引擎：主要结合JS实现html，一种以handlebars mustache为代表，实现方式为拼字符串。另一种以react为代表的，实现方式为VDOM。
服务端引擎：以velocity这种为代表，可结合Java等语言实现，由服务端生成html返回客户端。

模板引擎的使用情况

全前置、前后各半、全后模板。
前端模板会分担一部分服务器压力，但是也会给客户端增加压力，所以我们需要根据实际情况选择合适的方法。

最佳方法

不管是前端的还是后端的模板都有一定的局限性。最好是用一种后端语言，单纯提供渲染页面所需要的接口，Node中间层用模板引擎来渲染页面，使得页面直出。
这样就可以达到一处编写，到处运行的目的。
