TCP/IP基础

一、通过http举例来说明tcp/ip协议族进行网络通信时的过程，首先作为发送端的客户端在应用层发出一个想看某个Web页面的http请求。
接着为了传输方便，在传输层吧应用层处收到的数据进行分割，并在各个报文上打上标记序号及端口号转发给网络层。在网络层中，增加作为通信目的地的MAC地址后转发给链路层，这样一来发送网络的通讯请求就准备齐全了。到达服务端以后从链路层开始逐级往上，最终到达应用层，才算真正接收到http请。这其中为了识别每一次的内容，每一层的数据在处理的时候都会增加一个首部，而在接收端则是删除一个首部。
这就是封装。
二、负责传输的ip协议
ip位于网络层，是网际协议，其作用就是把各种数据包传送给对方，而这里需要很多条件，其中最重要的是条件时ip地址和mac地址，IP地址指明了节点被分配到的地址，mac地址是网卡所属的固定地址。ip可变mac基本不会变。
1.使用ARP（Address Resolution Protocol）解析地址协议。使用这个协议ip见可以通过AR中转实现通信。
2.在到达通信目标前的中转过程中，那些计算机和路由器等网络设备只能获悉很粗略的传输路线。这种机制称为路由机制。
三、确保可靠性的TCP协议
TCP位于传输层，提供可靠的字节流服务（为了方便传输，将大块数据分割成以报文段为单位的数据包进行管理）且TCP能够确认数据最终是否送达对方。为了确保数据能达到目标，TCP采用乱三次握手策略，TCP将数据包送出去以后，会一直向对方确认数据是否送达，握手过程中使用了TCP的flag（SYN和ACK）握手的过程如下，发送端首先发送一个带SYN标志的数据包给对方，接收端收到后，回传一个带有SYN/ACK标志的数据包以示传达确认，最后发送端再传回一个带有ACK标志的数据包以示握手结束。如果握手阶段出现错误。发送端在意同样的顺序在发送一次。
四、负责域名解析的DNS服务
DNS服务位于应用层，提供域名到ip地址之间的解析服务。也就是DNS协议通过域名查找IP地址或者逆向从IP地址反查域名的服务。
五、URI和URL
URI是Uniform Resource Identifier其中Uniform规定统一的格式可方便处理各种不同类型的资源，而不用根据上下文环境来识别资源指定的访问方式。Resource资源的定义是可标识的东西。
Identifier表示可标识的对象也就是标识符，综上URL就是由某个协议方案表示的资源的定位标识符。协议方案是指访问的资源所使用的协议类型的名称。URI用字符串标识某一互联网资源， 而URL标识资源的地点。
1.URI的格式
表示指定的URI，要使用涵盖全部必要信息的绝对URI，绝对的URL和相对URL。
绝对URI的基本格式：协议方案名+登录信息+服务器地址+服务器端口号+带层次的文件路径+查询字符串+片段标识符
六、简单的HTTP协议
HTTP协议用于客户端和服务器之间的通信，请求访问文本或图像疯资源的一端称为客户端，提供资源响应的一端称为服务器端。从客户端建立通信，服务器端在没接收到请求之前不会发送响应。
HTTP协议是不保存状态的协议，既无状态的协议，使用HTTP协议，每当有新的请求发送时，就会有的新的响应产生.引入Cookie技术，就是为了保持状态，有了Cookie之后就可以管理状态了。
HTTP协议使用URI来定位互联网上的资源，请URI的方式有多种， 用完整的URI和在首部字段Host写明网络域名或IP地址。
1.HTTP的方法

GET: GET方法用来请求访问已经被URI识别的资源。
POST: POST方法用来传输实体的主体。
PUT: PUT方法用来传输文件，但是此方法不安全，但是配合web应用程序的验证机制或者RESTful标准的同类web网站就可以的。
HEAD: 获得报文的首部，用于确认URI的有有效性及资源的更新的日期时间等。
DELETE: 删除文件。和put方法一致。
OPTIONS: 用来查询针对请求的URI指定的资源支持的方法。
TRACE: 让web服务器将之前的请求通信返回给客户端的方法，此方法可以查询发送出去的请求时怎样被加工修改的。容易引起XST（Cross-Site-Tracing，跨站追踪）
CONNECT: 此方法是要求在于代理服务器通信时简历隧道，实现用隧道协议进行TCP通信，主要使用SSL（Secure Sockets Layer，安全套接层）和TSL（Tansport Layer Security, 传输层安全）。

先说RESTful API的缺点

一、扩展性
随api的不断发展，会变得越来越臃肿。
server返回的用户信息越来越多，client需要的信息只是其中的一部分，这样会增加网络传输量。并且不利于client处理数据。
调用多个api才能满足不同的数据需求，导致需要调用不同的api很是不灵活。

GraphQL

简单来说就是取那些数据是由客户端说了，在REST中给，给那些数据是server决定的，client只能从中挑选。在GraphQL中client直接对server说想要什么数据，server就负责返回什么数据。

文档

文档是mongDB的核心，类似于SQLite数据库关系数据库中的每一行。多个键及其关联的值放在一起就是文档。

文档的逻辑关系

1.嵌入式关系
2.引用式关系，将两个文档分开，通过引用文档的_id
字段来建立关系。

集合

集合就是一组文档的集合，就相当于关系数据库中的表一样。
1.元数据
数据库的信息存储在集合中，他们使用统一的命名空间DBNAME.system.*

常用命令

use dbname 创建数据库
db 查看当前数据库
show dbs 查看所有数据库
db.dropDatabase() 销毁数据库
createCollection()：创建集合
db.COLLECTION.drop()：删除集合
db.COLLECTION_NAME.insert(document)：插入文档
db.COLLECTION_NAME.update(SELECTION_CRITERIA,UPDATED_DATA)：更新文档
db.COLLECTION_NAME.save({_id:ObjectId(),NEW_DATA})：替换已存在的文档
db.COLLECTION_NAME.remove(DELECTION_CRITERIA)：删除文档

查询语句

当find中传入多个键值对时，MongoDB就会将其作为AND处理
db.users.find() 查询数据，不带参数返回所有数据。
db.users.find().pretty() 可以使查询的结果更为美观。
OR处理方法

db.post.find(
{
$or: [
{key1: value1}, {key2:value2}
]
}
).pretty()
PS : {$gt:10} 表示大于10，另外，$lt 表示小于，$lte 表示小于等于，$gte 表示大于等于，$ne 表示不等于。

文档操作

创建集合 db.createCollection(“shiyanlou2”, { capped : 1, autoIndexID : 1, size : 6142800, max : 10000 } ) #带参数
{ “ok “: 1 }
参数描述：

  • capped：类型为 Boolean，如果为 true 则创建一个固定大小的集合，当其条目达到最大时可以自动覆盖以前的条目。在设置其为 true 时也要指定参数大小；
  • autoIndexID：类型为 Boolean，默认为 false，如果设置为 true，则会在 _id field.s 上自动创建索引；
  • size：如果 capped 为 true 需要指定，指定参数的最大值，单位为 byte；
  • max：指定最大的文档数。 在 Mongodb 中也可以不用创建集合，因为在创建文档的时候也会自动的创建集合。

    查询索引和聚合

    1.条件操作符
    MongoDB 中的条件操作符有：
    (>) 大于 - $gt #greater than
    (<) 小于 - $lt #less than
    (>=) 大于等于 - $gte #equal
    (<= ) 小于等于 - $lte
    2.条件操作符
    $type
    3.limit与skip
    读取指定数量的数据记录 limit
    读取时跳过指定数量的数据记录 skip
    当然，还可以添加 find 的查找条件的参数，以便进行更精确的查找。
    4.排序sort()
    升序用1表示，降序用-1表示
    5.索引ensureIndex()
    索引是特殊的数据结构，索引存储在一个易于遍历读取的数据集合中，索引是对数据库集合中一个文档或多个文档的值进行排序的一种结构。
    6.聚合 aggregate()

实例：db.shiyanlou.aggregate([{$group:{_id:”$name”, user:{$sum:”$user_id”}}}])
7.管道
MongoDB 的聚合管道将 MongoDB 文档在一个管道处理完毕后将结果传递给下一个管道处理。管道操作是可以重复的。
表达式：处理输入文档并输出。表达式是无状态的，只能用于计算当前聚合管道的文档，不能处理其它的文档。
聚合框架中常用的几个操作
$project：修改输入文档的结构。可以用来重命名、增加或删除域，也可以用于创建计算结果以及嵌套文档。
$match：用于过滤数据，只输出符合条件的文档。$match 使用 MongoDB 的标准查询操作。
$limit：用来限制 MongoDB 聚合管道返回的文档数。
$skip：在聚合管道中跳过指定数量的文档，并返回余下的文档。
$unwind：将文档中的某一个数组类型字段拆分成多条，每条包含数组中的一个值。
$group：将集合中的文档分组，可用于统计结果。
$sort：将输入文档排序后输出。
$geoNear：输出接近某一地理位置的有序文档。

高级索引和查询

为了验证我们使用了索引，可以使用 explain 命令
1.原子操作
所谓原子操作，就是要么执行成功，要么执行失败，执行成功完成既定任务，执行失败还原执行前的状态。
常用原子操作命令：
1) $set
用来指定一个键并更新键值，若键不存在并创建。
2) $unset
用来删除一个键。
3) $inc
$inc 可以对文档的某个值为数字型（只能为满足要求的数字）的键进行增减的操作。
4) $push
把 value 追加到 field 里面去，field 一定要是数组类型才行，如果 field 不存在，会新增一个数组类型加进去。
5) $pushAll
同 $push ,只是一次可以追加多个值到一个数组字段内。
6) $pull
从数组 field 内删除一个等于 value 值。
7) $addToSet
增加一个值到数组内，而且只有当这个值不在数组内才增加。
8) $pop
删除数组的第一个或最后一个元素。
9) $rename
修改字段名称
10) $bit
位操作，integer 类型
2.查询分析explain()
explain 操作提供了查询信息，使用索引及查询统计等。有利于我们对索引的优化
3.hint()
虽然 MongoDB 查询优化器一般工作的很不错，但是也可以使用 hints 来强迫 MongoDB 使用一个指定的索引。
通过这种方法在某些情形下会提升性能。
